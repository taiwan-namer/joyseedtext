# 商品管理區：首頁設定與階層位置說明

## 首頁設定的功用

**首頁設定**用來決定這門課程是否、以及如何出現在前台首頁：

- **不顯示**：僅在「課程介紹／課程列表」頁出現，不出現在首頁區塊。
- **顯示於首頁輪播**：可出現在首頁的輪播橫幅（需與「前台設定」的輪播項目連動）。
- **顯示於熱門課程**：可出現在首頁的「熱門課程」區塊。

目前商品管理區表格中的「首頁設定」欄位為預留，尚未與資料庫欄位連動。

## 階層位置的功用

**階層位置**用來控制課程在列表中的**顯示順序**（數字愈小愈前面）：

- 例如：階層 1 的課程會排在階層 2 之前。
- 可用於「熱門課程」或「課程列表」的排序，讓重要課程排在前面。

目前商品管理區表格中的「階層位置」欄位為預留，尚未與資料庫欄位連動。

---

## 建議：如何新增至「新增課程」表單

若要在**新增課程**時就能選擇首頁設定與階層位置，建議作法如下：

1. **資料庫**（Supabase `classes` 表）  
   - 新增欄位（可選）：  
     - `show_on_home`（text 或 enum）：`none` | `carousel` | `featured`  
     - `sort_order`（integer）：數字愈小愈前面，預設可為 `0` 或 `999`  

2. **新增課程表單**（`CreateClassForm` / 新增課程頁）  
   - **首頁設定**：下拉選單，選項例如  
     - 不顯示  
     - 顯示於首頁輪播  
     - 顯示於熱門課程  
   - **階層位置**：數字輸入框（整數），預設值例如 `0`  

3. **建立課程的 API / Action**（`createClass` 或 `createCourseFull`）  
   - 表單送出時一併寫入 `show_on_home`、`sort_order`。  

4. **前台首頁與課程列表**  
   - 讀取課程時依 `show_on_home` 篩選要出現在輪播／熱門區的課程。  
   - 依 `sort_order` 排序（asc）。  

完成以上步驟後，新增課程時即可直接選擇首頁設定與階層位置，無需事後在商品管理區手動調整。
